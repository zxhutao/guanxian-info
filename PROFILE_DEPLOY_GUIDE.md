## 🚀 部署步骤

### 第一步：部署云函数

在**微信开发者工具**中操作：

#### 1. 部署 `updateProfile` 云函数
```
位置：uniCloud-tcb/cloudfunctions/updateProfile/
操作：右键文件夹 → 「创建并部署：云端安装依赖」
```

#### 2. 部署 `bindPhone` 云函数
```
位置：uniCloud-tcb/cloudfunctions/bindPhone/
操作：右键文件夹 → 「创建并部署：云端安装依赖」
```

### 第二步：创建数据库集合
手机绑定功能需要一个 `verifications` 集合来存储验证码记录，系统会**自动创建**。

如果需要手动创建，可以执行以下步骤：
1. 打开云数据库控制台
2. 创建集合：`verifications`
3. 集合权限设置：
   - 创建：仅创建者
   - 读取：仅创建者
   - 更新：仅创建者
   - 删除：任何人

### 第三步：重启小程序
1. 关闭微信开发者工具
2. 重新打开项目
3. 点击「编译」

## 📱 使用测试流程

### 测试1：页面跳转
1. 登录小程序
2. 进入「我的」页面
3. 点击用户信息区域（特别是右边箭头）
4. 应该成功跳转到「个人信息」编辑页面

### 测试2：个人信息编辑
1. **修改昵称**：输入新昵称 → 保存 → 页面返回 → 昵称应更新
2. **更换头像**：选择图片 → 保存 → 头像应更新
3. **切换用户类型**：选择其他类型 → 保存 → 类型应更新

### 测试3：手机号绑定
#### 绑定新手机号
1. 点击「绑定」按钮
2. 输入手机号 → 获取验证码
3. 查看控制台输出：`模拟发送短信到 ${phone}: 验证码 ${code}，有效期5分钟`
4. 输入验证码 → 绑定成功

#### 更换手机号
1. 已有手机号状态下点击「更换」
2. 输入新手机号 → 验证码流程同上
3. 手机号应更新

#### 解绑手机号
1. 已有手机号状态下点击「更换」
2. 选择「解绑手机号」
3. 确认解绑 → 手机号应清空

## 🔧 技术细节

### 云函数返回格式
```javascript

  data: { ... }
}

// 失败

  message: '错误信息'

```

### 验证码存储
- 集合：`verifications`
- 字段：`phone`, `code`, `userId`, `status`, `createdAt`, `expiresAt`
- 有效期：5分钟
- 自动清理：每次发送验证码时会清理过期的记录

### 手机号绑定限制
1. **格式验证**：必须是 11 位中国大陆手机号
2. **唯一性检查**：不能与其他用户重复绑定
3. **频率限制**：验证码过期后才能重新发送

## 🧪 注意事项

### 实际短信服务
当前的实现是**模拟短信发送**，实际使用需要：
1. 注册腾讯云短信服务
2. 申请短信签名和模板
3. 在代码中替换 `sendSMSCode` 函数为真实短信发送
4. 配置云函数环境变量

### 头像上传
目前头像上传使用的是本地临时路径，实际使用需要：
1. 配置云存储
2. 实现图片上传到云存储
3. 获取云存储 URL 地址

### 生产环境建议
1. **增加频率限制**：防止恶意发送验证码
2. **使用Redis**：验证码缓存更高效
3. **记录日志**：绑定/解绑操作审计日志
4. **二次确认**：重要操作增加二次确认

## 📞 问题排查

### Q1：页面无法跳转
检查：
- `pages.json` 中是否有 `pages/user/profile` 注册
- `pages/user/index.vue` 的 `goLogin` 方法是否修改成功
- 路径是否正确

### Q2：云函数调用失败
检查：
- 云函数是否部署成功
- 函数名是否正确：`bindPhone`, `updateProfile`
- 参数格式是否正确（参考本文档）

### Q3：验证码无效
检查：
- 查看控制台输出的验证码
- 验证码是否过期（5分钟）
- 数据库 `verifications` 集合是否有数据

### Q4：手机号绑定失败
检查：
- 手机号格式是否正确
- 手机号是否已被其他用户绑定
- 用户ID是否正确
